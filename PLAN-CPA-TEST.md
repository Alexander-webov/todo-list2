# План: тест CPA-монетизации (вместо подписки)

Цель теста — **за пару тысяч рублей** узнать одну цифру: сходится ли арбитраж.
То есть **доход с посетителя (RPV)** больше или меньше, чем **цена клика (CPC)**.
Если RPV > CPC — у тебя бизнес, который масштабируется платным трафиком.
Если меньше даже на точной аудитории и точных офферах — честный сигнал закрывать.

## Что нужно сделать тебе — по шагам

### Шаг 1. Зарегайся в CPA-сетях (1 день)
Нужны офферы под русских фрилансеров. Сети: **Admitad, Leads.su, ePN, Where.Money**.
Подключи 2–3 оффера:
- **Обучение** — курсы (Skillbox, Нетология, GeekBrains). Платят за лид/заявку.
- **Финансы** — счёт/карта для самозанятых (Тинькофф, Точка, Альфа). Самые жирные выплаты.
- (опц.) **Сервис/инструмент** для фрилансеров.

Запиши для каждого: партнёрскую ссылку и размер выплаты (₽ за действие).

### Шаг 2. Добавь офферы через админку (15 минут)
Заходишь в свою админку → раздел рекламы → добавляешь объявление:
- заголовок, описание, картинку (как на твоём скриншоте);
- в поле ссылки — **партнёрскую ссылку CPA с твоим subId уже внутри** (чтобы сеть атрибутировала конверсии этому сайту);
- позиция: `feed` (в ленту) и/или `sidebar`.
Никаких правок кода — всё через панель.

### Шаг 3. Заведи цель в Яндекс.Метрике (5 минут)
Метрика → Цели → JavaScript-событие с идентификатором **`ad_click`**.
Эта цель покажет, сколько людей кликают по рекламе (CTR). Серверный счётчик кликов у тебя и так есть.

### Шаг 4. Задеплой (5 минут)
Залей на Railway как обычно. Проверь на сайте: лента открыта без регистрации, между карточками видны офферы, клик уводит на партнёрку.

### Шаг 5. Купи тестовый трафик (бюджет ~2 000 ₽)
Пригони **300–500 целевых** русских фрилансеров — это твоя арбитражная игра. Веди их на главную или на категорию (`/category/...`).
Помечай источник UTM-меткой, чтобы потом отделить от органики.

### Шаг 6. Посчитай результат (через 3–7 дней — у CPA-сетей лаг по конверсиям)
Собери три числа:
1. **Посетители из теста** — Яндекс.Метрика (по UTM).
2. **Потрачено на трафик** — из рекламного кабинета. CPC = потрачено / посетители.
3. **Доход** — из дашборда CPA-сети (подтверждённые конверсии × выплата).

Считай:
```
RPV (доход с посетителя) = доход CPA / посетители
Вердикт:  RPV > CPC  →  арбитраж сходится, можно лить платный трафик и масштабировать
          RPV < CPC  →  не сходится; органика (SEO/сообщества) или закрывать
```

## Заранее назначь порог решения (чтобы не утонуть)

- Если RPV хотя бы близко к CPC (в пределах ~30%) — есть с чем работать: крути офферы, плейсменты, креативы.
- Если RPV в разы ниже CPC даже на точной аудитории — это потолок модели. Не уговаривай себя «ещё чуть-чуть докручу». Тогда: либо только бесплатный трафик (SEO вдолгую), либо закрывать и применить урок к следующему проекту.

Решение принимаешь по цифрам, а не по вложенному времени.
